Θέλω να φτιάξω μία web εφαρμογή παρουσίασης έργων της Περιφέρειας Ηπείρου. 
Τη βασική δουλειά στο webapp θα την κάνει ένας llm agent.
(κάνε έρευνα για να καταλάβεις τι αρχεία μπορεί να περιέχει ένα έργο της Περιφέρειας Ηπείρου, δημόσια έργα γενικότερα.)
Ο agent θα παίρνει τον φάκελο με τα αρχεία του έργου, θα τα διαβάζει, επειτα θα φτιάχνει ένα markdown αρχειο με όλη την 
πληροφορία απο τα αρχεία που διάβασε. Έπειτα ο llm agent θα ορίζει το βελτιστο workflow για τη δημιουργία ενός website (που
το φτιάχνει ο agent) που εκεί θα γίνεται παρουσίαση με μοντέρνο UI του έργου που έχει επιλέξει ο χρήστης. 
Γενικότερα όλο το project θέλω να δουλεύει με queries στο llm. Ουσιαστικά το project αποτελείται από 2 βασικά βήματα. Αρχικά 
την αποδέλτηση του περιεχόμενου του φακέλου του έργου που επιλέγει ο χρήστης πως θέλει να παρουσιάσει και έπειτα την παρουσίαση
του content σε ένα website που θα δημιουργεί ο agent.


Το website θα ειναι ένα στατικό website μιας κύριας σελίδας και θα εχει ένα μενού που θα λειτουργεί με anchors για να 
γίνεται η πλοήγηση στην σελίδα.

Θέλω να δινω εναν φάκελο με αρχεία από ένα έργο (το webapp θελω να κάνει την ίδια δουλειά πάντα ανεξάρτητα από το έργο 
που θα του δώσω).

Φτιάξε μου ένα Project Requirement Document για την υλοποίηση του παραπάνω project. Θέλω το PRD να μου το δώσεις σε ένα 
.txt αρχείο.

Το βασικό στοιχείο του project θα είναι ο llm agent να μπορεί να διαβάζει τα πολυμεσικά αρχεία του φακέλου του έργου και να φτιάχνει μόνος του 
την δομή και γενικά ολόκληρο το website, (πχ αν έχει πολλές εικόνες να διαμορφώνει κατάλληλα το website). Σε γενικά πλαίσια θέλω ο agent να μπορεί να καταννοεί 
πλήρως το έργο και να φτιάχνει ένα κατάλληλο website παρουσίασης του έργου.
